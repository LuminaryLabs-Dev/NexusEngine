# Core Object Fidelity Domain

`n:object:fidelity` preserves a Core Object's recognizable identity through several valid forms under changing visual and performance constraints.

## Owns

- fidelity profiles
- source-derived and view-derived forms
- form builders and builds
- visible and complete readiness
- capture dependencies
- active and pending fidelity packages
- contextual adaptation
- stale-build rejection
- atomic package replacement

## Does not own

- Core Object identity
- tree morphology, creature anatomy, or object-specific generation
- renderer objects or GPU resources
- capture rendering
- asset transport
- WebGPU, Three.js, DOM, or Canvas behavior

## Contract

```txt
Core Object defines what the object is.
Object Fidelity prepares valid forms that preserve it.
Capture records appearance when a form needs observed views.
The active package remains usable until a complete replacement commits.
```

Built-in reference form builders support source forms, capture-derived forms, and absent forms. Object-specific builders remain external.
