## Root Lesson
- id: deep-bug-root-2026-06-23-runtime-kit-definition-immutability
- statement: `defineRuntimeKit()` needs nested definition immutability before RuntimeKit, composer, DSK, and proof surfaces can trust a kit object as stable after validation.
- why it matters: Current smoke tests pass, but live probes show frozen RuntimeKit objects still expose mutable systems, provider tokens, and requirements; callers can inject systems before install, forge composer provider tokens, and mutate installed kit definitions so `engine.kits` no longer matches scheduler state.
