## Root Lesson
- id: domain-kit-idea-root-2026-06-19-0100
- statement: The next useful expansion layer is accepted mutation and time/config policy: validate service calls before side effects, emit completion once, catch up large time deltas, and reject or normalize invalid config before simulation state changes.
- why it matters: Neighboring bug evidence shows that broad DSK promotion is blocked not only by terminal-state semantics, but also by service APIs that mutate before acceptance, repeated completion emissions, one-step time advancement, and non-finite config entering state.

  lesson: Completion idempotency should be tracked as a policy kit because proof, objective, and lifecycle surfaces need one-shot completion receipts rather than repeated completed-state emissions.
  evidence: Live `src/objective-flow-kit.js` emits `ObjectiveFlowCompleted` whenever state remains completed, and the deep bug packet recorded repeated completion batches.
  look further: Decide whether reset belongs with completion idempotency or remains a Replicated State Domain responsibility.
- id: simulation-time-policy-domain-2026-06-19-0100
  parent: domain-kit-idea-root-2026-06-19-0100
  lesson: Simulation Time Policy Domain should own large-delta subdivision, leftover progress, scale rules, and multi-cycle catch-up across transport, schedule, lifecycle, and scenario services.
  evidence: Deep bug packet `2026-06-19T00-54-03-0400` found a transport carrier with enough delta for multiple stops only advances one stop, while live `src/transport-route-kit.js` consumes `progress >= 1` once per tick.
  look further: Decide whether time catch-up is a ProtoKit policy service, a core clock/scheduler primitive, or a combination.
- id: config-normalization-domain-2026-06-19-0100
  parent: domain-kit-idea-root-2026-06-19-0100
  lesson: Config Normalization Domain should own finite-number checks, enum/default normalization, and validation reports before authored or restored datasets enter deterministic simulation state.
  evidence: Live `src/schedule-kit.js` stores `Number(config.scale ?? 1)` directly, and the deep bug packet found `scale:"fast"` corrupts elapsed time as `NaN`.
  look further: Define fail-fast versus clamped-default policy before adding more authored dataset imports.
