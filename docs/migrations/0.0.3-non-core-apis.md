# 0.0.3 Non-Core API Migration

NexusEngine removed optional and game-specific exports without forwarding
aliases.

## Fishing

Before:

```js
import { createFishingKit, createFishingRenderer } from "nexusengine";
```

After:

```js
import {
  createFishingKit,
  createFishingRenderer
} from "@luminarylabs/nexusengine-kits/fishing-kit";
```

Fishing realism and shaders use:

```js
import { createFishingRealismKit } from "@luminarylabs/nexusengine-kits/fishing-kit/realism";
import { fishingShaders } from "@luminarylabs/nexusengine-kits/fishing-kit/shaders";
```

## Optional Gameplay Factories

AR, interaction, light combat, companion command, camera collision, character
ragdoll, forest placement, objective, spatial room, greybox building, surface
placement, interaction target, collectible, symbol alignment, sorting, reveal
light, moving target, lock-and-socket, and render-descriptor factories now use:

```js
import {
  createARKit,
  createInteractionKit,
  createObjectiveKit
} from "@luminarylabs/nexusengine-kits/migrated-gameplay";
```

Prefer the individual trusted-registry manifest and installer path when
assembling applications dynamically.

## Complete Games

`createReefRescueKit`, `createShrinePuzzleKit`, `createCorruptionWorldKit`,
`createTreeRunnerKit`, and `createMicroPlatformerKit` are not package exports.
Consume their game repositories directly.

## Renderers And Hooks

Core no longer provides fishing-specific renderer modes, shaders, realism
profiles, terrain bindings, or sequence-controller coupling. Supply external
adapters through public runtime-kit, DSK, terrain, and sequence contracts.

## Required Proof

- the replacement installs without private NexusEngine imports
- migrated behavior passes focused and composed tests
- no NexusEngine public entrypoint reaches the old implementation
- the application imports only public package subpaths
